# 블로그 글쓰기 가이드

## 글 말투/스타일

- **해체 (반말)** 사용. 경어체 X
- 기술 블로그지만 딱딱하지 않게. 개발 경험을 이야기하듯이
- "~했다", "~이다" 체
- 1인칭 시점, 경험 기반 서술
- 과장 없이 솔직하게 (삽질도 그대로 씀)

### 좋은 예시

```
SQLite를 선택한 이유는 명확했다. MVP에 PostgreSQL은 과하다.
```

```
물론 이게 끝이 아니었다. 승률은 처참했고, BTC 하나만으로는 기회가 너무 적었다.
```

### 안 쓰는 표현

- "~합니다", "~입니다" (경어체)
- "여러분", "독자 여러분" (독자에게 말 거는 톤)
- 이모지 남발

## 글 구조

1. **도입** — 왜 이걸 했는지, 문제 상황 (1~2문단)
2. **본문** — 기술적 내용을 h2 섹션으로 나눠서. 코드 스니펫 포함
3. **마무리** — 결과 + 다음 글 예고 (시리즈인 경우)
4. **참조 이슈** — `<hr>` 후 GitHub 이슈 링크 목록

## HTML 포맷

- 순수 HTML 태그만 사용 (`<h2>`, `<p>`, `<ul>/<li>`, `<pre><code>`, `<strong>`, `<em>`, `<a>`, `<hr>`)
- `<h1>` 사용 안 함 (티스토리가 제목을 h1으로 렌더링)
- 코드 블록: `<pre><code class="language-python">...</code></pre>`
- 글자 수: 2000~3500자 (HTML 태그 포함)

## 참조 이슈 형식

```html
<hr>
<p><em>참조 이슈:</em>
<a href="https://github.com/SeungMinK/cryptobot/issues/1">#1 프로젝트 초기 설정</a> ·
<a href="https://github.com/SeungMinK/cryptobot/issues/4">#4 API Key 발급</a>
</p>
```

## 태그 규칙

- 쉼표로 구분: `CryptoBot,자동매매,업비트,Python,개발일지`
- 프로젝트명 항상 포함
- `개발일지` 항상 포함
- 핵심 기술 키워드 3~5개

## 시리즈 연결

- 각 글 마지막에 다음 편 예고
- 마지막 편은 시리즈 마무리 + 회고
